# Задание на реструктуризацию таблицы запланированных заявок (на завтра)

**Область применения:** ТОЛЬКО таблица запланированных заявок (`#requestsPlanningTable`) на вкладке планирования. Основная таблица заявок остается без изменений.

## Список задач:
1. **Перенос чекбоксов:**
   - Переместить чекбоксы в первую колонку таблицы.
   - Чекбокс должен располагаться строго **под порядковым номером** строки (а не в одной строке с ним), чтобы сохранить компактность.

2. **Перенос кнопки редактирования:**
   - Кнопку редактирования заявки (иконка карандаша) перенести из колонки действий в колонку с **комментариями**.
   - Расположить её справа от кнопки "Все комментарии" (или кнопки открытия модалки комментариев).

3. **Удаление колонки "Действия":**
   - После переноса чекбокса и кнопки редактирования, полностью удалить крайнюю правую колонку "Действия" (Actions) для освобождения экранного пространства.

4. **Синхронизация с JS:**
   - Учесть, что строки таблицы планирования генерируются динамически в `public/js/handler.js`. Необходимо обновить HTML-шаблон строки в коде.
