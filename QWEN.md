# QWEN.md — Рекомендации для проекта lan-install.online

## Деплой проекта

Для деплоя используется скрипт `git-deploy` в корне проекта.

### Использование

```bash
./git-deploy "Ваш комментарий к коммиту"
```

### Что делает скрипт

1. Проверяет статус репозитория (`git status`)
2. Добавляет все изменения (`git add .`)
3. Создаёт коммит с указанным комментарием (`git commit -m "..."`)
4. Выполняет push в удалённый репозиторий (`git push`)
5. Подключается к серверу по SSH и выполняет `git pull` в директории `/var/www/lan-install`
6. Очищает кеш Laravel:
   - `php artisan cache:clear`
   - `php artisan view:clear`
   - `php artisan config:clear`

### Требования

- SSH-ключ: `~/.ssh/git_lan_install`
- SSH-хост: `lan-install` (настроен в `~/.ssh/config`)
- Директория на сервере: `/var/www/lan-install`
- Лог-файл: `$HOME/logs/git-deploy.log`

### Формат комментариев к коммитам

Проект следует соглашению [Conventional Commits](https://www.conventionalcommits.org/):

```
<type>: <description>

[optional body]
```

**Типы коммитов:**
- `feat` — новая функциональность
- `fix` — исправление ошибки
- `docs` — изменения в документации
- `style` — форматирование, стили (без влияния на логику)
- `refactor` — рефакторинг кода
- `test` — добавление или исправление тестов
- `chore` — изменения в сборке, зависимостях, инструментах
- `security` — исправления уязвимостей, ограничения доступа

**Примеры:**
```bash
./git-deploy "feat: добавить экспорт отчётов в Excel"
./git-deploy "fix: исправить ошибку валидации email"
./git-deploy "security: ограничить доступ к типам заявок только для админов"
```

### Важные замечания

1. **Комментарий обязателен** — скрипт требует передачи строки с комментарием
2. **Очистка кеша выполняется локально** — команды `php artisan cache:clear` выполняются на локальной машине, а не на сервере
3. **Логирование** — все этапы деплоя записываются в лог-файл `$HOME/logs/git-deploy.log`
4. **Прерывание при ошибке** — скрипт использует `set -e`, любая ошибка останавливает выполнение

## Технологии проекта

- **Backend:** Laravel 10+
- **Frontend:** Vanilla JavaScript, Bootstrap 5
- **База данных:** PostgreSQL
- **Карты:** Яндекс.Карты API

## Структура проекта

```
lan-install.online/
├── app/Http/Controllers/     # Контроллеры
├── app/Http/Middleware/      # Middleware
├── resources/views/          # Blade-шаблоны
├── public/js/                # JavaScript файлы
├── routes/web.php            # Маршруты
└── git-deploy                # Скрипт деплоя
```

## Права доступа

Система использует роли:
- `admin` — полный доступ ко всем функциям
- `user` — ограниченный доступ (просмотр, создание заявок)
- `fitter` — доступ монтажника

Проверка прав выполняется через middleware `AddUserRolesToView`.
