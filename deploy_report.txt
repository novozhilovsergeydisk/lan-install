Отчет об изменениях (Фикс залипающей шапки таблицы планирования):
- Файл: public/css/custom.css
- Проблема: При прокрутке страницы все заголовки <th> шапки таблицы выходили из нормального потока из-за свойства `position: fixed` и накладывались друг на друга, в результате чего пользователь видел только последний элемент DOM-дерева шапки (колонка "Бригада"). Жестко заданный `top: 135px !important` также переопределял динамически вычисляемые значения из JS-скрипта sticky-fix.js.
- Решение: Заменено `position: fixed !important` на `position: sticky !important` для класса `.is-sticky` у шапки таблицы. Свойство `top` удалено из CSS, чтобы работал динамический расчет из JS.
- Регрессии: Риск минимален. Свойство `position: sticky` является современным стандартом для залипающих заголовков таблиц и не нарушает ширину столбцов.
- Безопасность: Изменения касаются только визуального отображения (CSS), безопасность не затронута.
