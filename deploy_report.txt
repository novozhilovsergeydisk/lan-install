Отчет перед деплоем:

1. Логика (Logic):
   - Исправлена ошибка 500 на сервере при запросе метрик системы.
   - Проблема заключалась в том, что бинарный файл утилиты `sys-monitor` был скомпилирован на macOS (в моем окружении) и добавлен в Git. При запуске на Linux-сервере он не мог быть исполнен, из-за чего функция `shell_exec()` возвращала `null`, а контроллер отдавал 500 ошибку.
   - Файл `utils/C/sys-monitor/sys-monitor` удален из индекса Git и добавлен в `.gitignore`.
   - В скрипт `./update-bot` добавлена команда для автоматической компиляции `sys-monitor` непосредственно на сервере.
   - Скрипт `./update-bot` был успешно запущен, утилита перекомпилирована на Linux, и теперь API `system/metrics` на сервере работает корректно.

2. Регрессии (Regressions):
   - Регрессий нет. Логика самого мониторинга не изменилась. Исключение бинарника из Git - правильная архитектурная практика.

3. Безопасность (Security):
   - Чувствительных данных не добавлено.
